## 1. Research & Discovery
- Analyze the needs of both user types (Idea Owner and Investor).
- Map out the journey for each persona to understand goals and expectations.
- Define Personas:
  - Idea Owner: Wants to showcase their project, connect with investors, and secure support.
  - Investor: Wants to discover promising ideas, review clear details, and evaluate risks and opportunities.

---

## 2. Information Architecture
- Create separate User Flows for each interface.
- Build a Site Map that includes:
  - Idea Owner pages: Submit project, track project status, communicate with investors.
  - Investor pages: Browse projects, filter by industry/stage, connect with idea owners.

---

## 3. Wireframing
- Develop Low-Fidelity Wireframes for both interfaces.
- Focus on:
  - A simple and guided submission process for idea owners.
  - Easy browsing and filtering for investors.
- Review flows with the team before moving to visual design.

---

## 4. UI Design (Visual Layer)

### ðŸŽ¯ Idea Owner Interface
- Dashboard showing project status (under review, published, in discussion with investors).
- Project Submission Form with clear steps (basic info, business plan, supporting files).
- Notifications for investor interest or new messages.
- Visual identity that communicates creativity and innovation.

### ðŸ’¼ Investor Interface
- Explore Page to browse projects with filters (industry, stage, funding amount).
- Project Cards with quick summaries (title, short description, stage, funding request).
- Project Details Page including:
  - Full description.
  - Business plan.
  - Team information.
  - "Contact Idea Owner" button.
- Favorites / Watchlist to save interesting projects.
- Visual identity that communicates trust and professionalism.

---

## 5. Prototyping & Testing
- Build an Interactive Prototype using Figma.
- Conduct Usability Testing with real users (students/innovators + investors).
- Collect feedback and refine the UI/UX accordingly.

---

## 6. Handoff to Development
- Prepare a Design System (colors, typography, buttons, components).
- Deliver Figma files with Design Specs and assets.
- Collaborate closely with developers to ensure accurate implementation.

---

## 7. Iteration & Improvement
- After launch: gather user feedback and monitor performance.
- Continuously improve both interfaces for better usability.
- Add new features based on market and user needs.
